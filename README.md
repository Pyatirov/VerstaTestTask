# Тестовое задание на вакансию Разработчика C#

Тестовое веб-приложение для приёмки заказов на доставку: создание заказа, просмотр списка всех заказов и детальный просмотр отдельного заказа.

## Стек технологий

**Backend**
- ASP.NET 9 (Web API, контроллеры)
- Entity Framework Core
- SQLite

**Frontend**
- React 18 + TypeScript
- Vite
- React Router DOM (роутинг между страницами)
- React Hook Form (валидация форм)
- Axios (HTTP-клиент)

## Функциональность

1. **Создание заказа** — форма со всеми обязательными полями: город и адрес отправителя, город и адрес получателя, вес груза, дата забора груза. Валидация на клиенте (React Hook Form) и на сервере (Data Annotations).
2. **Список заказов** — таблица со всеми созданными заказами, включая автоматически сгенерированный номер заказа (на основе `Id`).
3. **Просмотр заказа** — открывается по клику на строку в списке, отображает все поля заказа в режиме только для чтения.

## Требования для запуска

- [.NET 9 SDK](https://dotnet.microsoft.com/download)
- [Node.js](https://nodejs.org) (LTS-версия, 20+) с npm
- `dotnet-ef` (EF Core CLI, устанавливается один раз глобально):
  ```bash
  dotnet tool install --global dotnet-ef
  ```

## Запуск backend

1. Перейти в папку API-проекта:
   ```bash
   cd DeliveryOrders.Api
   ```

2. Восстановить зависимости NuGet (если не подхватилось автоматически при открытии в IDE):
   ```bash
   dotnet restore
   ```

3. Применить миграции и создать базу данных SQLite (файл `deliveryorders.db` появится в папке проекта):
   ```bash
   dotnet ef database update
   ```
   > Миграции также применяются автоматически при каждом запуске приложения (см. `Program.cs`), так что этот шаг можно пропустить — но выполнить его вручную полезно, чтобы сразу увидеть возможные ошибки конфигурации.

4. Запустить сервер:
   ```bash
   dotnet run
   ```
   Либо через Visual Studio — открыть `DeliveryOrders.sln` и нажать **F5** / **Ctrl+F5**.

5. Backend поднимется на `https://localhost:{порт}` — конкретный порт будет показан в консоли при запуске (задаётся в `Properties/launchSettings.json`).

## Запуск frontend

1. Перейти в папку клиента:
   ```bash
   cd delivery-orders-client
   ```

2. Установить зависимости:
   ```bash
   npm install
   ```

3. Проверить, что в `src/api/orders.ts` указан правильный адрес backend (`API_BASE_URL`) — должен совпадать с портом, на котором реально поднялся API из предыдущего шага:
   ```ts
   const API_BASE_URL = 'https://localhost:{ваш_порт}/api';
   ```

4. Запустить dev-сервер:
   ```bash
   npm run dev
   ```

5. Приложение откроется на `http://localhost:5173`.

## Порядок запуска

Backend необходимо запустить **до** обращения к нему с фронта (иначе запросы будут падать с сетевой ошибкой). Рекомендуемая последовательность:

1. Запустить backend (`dotnet run` или через Visual Studio)
2. Убедиться, что консоль показывает `Now listening on: https://localhost:...`
3. Запустить frontend (`npm run dev`)
4. Открыть `http://localhost:5173` в браузере

## Примечания по реализации

- **CORS** настроен в `Program.cs` на приём запросов конкретно с `http://localhost:5173` (адрес dev-сервера Vite по умолчанию) — при смене порта фронта нужно поправить и эту настройку.
- **Номер заказа** формируется на основе автоинкрементного `Id` записи в базе данных — это гарантирует уникальность номера силами самой СУБД, без дополнительной логики генерации и без риска коллизий при параллельных запросах.
- **База данных** — SQLite выбрана для простоты запуска тестового задания (файл базы создаётся локально, не требует установки отдельного сервера БД). Строка подключения — в `appsettings.json`, ключ `ConnectionStrings:DefaultConnection`.
